# Casos de Uso

Los casos de uso describen las principales interacciones entre los actores y el sistema.

## CU-001 Iniciar sesión

Permite autenticar usuarios en el sistema.

## CU-002 Consultar disponibilidad

Permite visualizar las fechas y horarios disponibles de las zonas comunes.

## CU-003 Solicitar reserva

Permite a un residente registrar una solicitud de reserva.

## CU-004 Aprobar reserva

Permite a la administración aprobar solicitudes.

## CU-005 Rechazar reserva

Permite a la administración rechazar solicitudes.

## CU-006 Cancelar reserva

Permite cancelar una reserva.

## CU-007 Gestionar zonas comunes

Permite crear, modificar o deshabilitar zonas comunes.

## CU-008 Configurar horarios

Permite definir horarios disponibles para cada zona.

## CU-009 Consultar historial

Permite visualizar las reservas realizadas.

## CU-010 Consultar reservas aprobadas

Permite a vigilancia verificar las reservas activas.
